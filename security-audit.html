Top 10 — Priority Actions

✓ Strip API key from Hono access logs
Key is passed as ?key=mem_xxx query param on /auth/done — logged by logger() middleware on every request. Anyone with log access sees all keys ever issued. Use POST + session or one-time token instead of URL param.
✓ Age private key written to /tmp on every MCP decrypt
src/routes/mcp.ts writes the user's age key (from X-Age-Key header) to /tmp/membridge-age-*.key before spawning the subprocess. Any local process or world-readable /tmp exposes it. Pipe key via stdin instead.
✓ No security headers on HTTP responses
Caddy config for membridge.yaro.fr is a bare reverse_proxy with no header directives. Missing: X-Frame-Options, X-Content-Type-Options, Content-Security-Policy, Strict-Transport-Security. Add them to the Caddyfile block.
✓ No firewall — all ports open by default
VPS has iptables policy ACCEPT with no rules. Port 8080 is also exposed publicly (another service). Install ufw or nftables: allow 22, 80, 443 only. Port 8080 should not be public.
✓ No fail2ban or SSH brute-force protection
SSH on default port 22, open to the world, root login allowed (key-only but still). No rate limiting or ban mechanism for failed attempts. Install fail2ban with SSH jail, or restrict SSH to a non-standard port + allowlist IPs.
✓ OAuth: redirect_uri not validated against registered values
src/routes/oauth-meta.ts — /oauth/authorize issues auth codes to any redirect_uri supplied, without checking it against the registered URIs in oauth_clients. Classic authorization code interception attack. Validate strictly before issuing code.
✓ No CSRF protection on state-mutating form POSTs
/auth/revoke (key deletion) and /oauth/authorize (OAuth consent) accept POST with no CSRF token and no sameSite cookie attribute. A malicious page can silently revoke the user's API key. Add sameSite: "Lax" to all cookies and a CSRF token to both forms.
✓ MCP route has no rate limiting
The /mcp POST endpoint (tool calls: decrypt, encrypt, search) has no rate limit middleware. The 100 req/hr limit only applies to /api/context. Decrypt operations spawn age subprocesses — easy to DoS. Apply rateLimitMiddleware or a dedicated stricter limit.
✓ PasswordAuthentication likely on by default (SSH)
/etc/ssh/sshd_config has #PasswordAuthentication yes — commented out, meaning the PAM default applies (typically yes). Explicitly set PasswordAuthentication no to force key-only login.
✓ API key visible in browser history + HTML form
Key appears in the URL (/auth/done?key=...) → browser history, referrer headers, server logs. Also embedded in an HTML hidden input for the revoke form — visible in devtools. Refactor to one-time server-side session or signed short-lived token.

Critical

✓ API key logged via query param on /auth/done
Resolved
Hono logger() logs every request URL. GET /auth/done?key=mem_xxx&user=y lands in stdout/container logs with the raw API key. Anyone reading Docker logs, log aggregators, or future log shipping gets every key ever generated.
src/routes/auth.ts:101–104
✓ Verified: Implemented one-time server-side done_tokens.Fix: POST key to short-lived signed session, redirect to /auth/done with a token, display once from session.
✓ Age private key written to /tmp in plaintext
Resolved
Every MCP tool call writes the user's age private key to /tmp/membridge-age-{uuid}.key. The file is removed after the subprocess exits, but: (1) /tmp is typically world-listable, (2) crash before cleanup leaves key on disk, (3) other processes on the host can race to read it. This is the master decryption key for the user's context.
src/routes/mcp.ts:12–51
✓ Verified: Spawns age processes with key passed directly over stdin (-i - and age-keygen -y).Fix: pass key via stdin to age -d -i /dev/stdin — no disk write needed.
✓ OAuth redirect_uri not validated against registered URIs
Resolved
The /oauth/authorize handler issues authorization codes redirecting to whatever redirect_uri the client supplies, without comparing to the stored oauth_clients.redirect_uris. Attacker registers a client, then sends victim to authorize with attacker's URI — code delivered to attacker, exchanged for token.
src/routes/oauth-meta.ts:70–124
✓ Verified: Validates redirect_uri matches client's registered URIs in both GET and POST endpoints.Fix: SELECT redirect_uris FROM oauth_clients WHERE client_id = ?; reject if supplied URI not in list.
